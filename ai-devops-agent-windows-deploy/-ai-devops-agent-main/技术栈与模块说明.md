# AI DevOps Agent — 技术栈、知识点与模块说明

本文面向开发与使用者，汇总项目涉及的技术要点、目录/包职责及常见使用方法，便于学习与二次开发。

---

## 一、技术栈与知识点

### 1. 后端运行时与框架

| 技术 | 用途 | 可延伸知识点 |
|------|------|----------------|
| **Java 17** | LTS 语言版本，记录、枚举、模块化依赖管理 | JVM 编码、JDK 选型 |
| **Spring Boot 3.2.x** | 嵌入式 Tomcat、自动配置、Starter 组装 | `@ConfigurationProperties`、生命周期、Profile |
| **Spring Web (MVC)** | REST：`@RestController`、`@RequestMapping` | HTTP 语义、DTO、统一封装 `Result` |
| **Spring Validation**（可选扩展） | 请求体验证注解 | `@Valid`、`BindingResult` |
| **Jackson** | JSON 序列化（实体 ↔ API） | 日期格式、枚举、注解定制 |

### 2. 数据持久化

| 技术 | 用途 | 可延伸知识点 |
|------|------|----------------|
| **MyBatis 3.x** | SQL 映射，本项目以 **`@Mapper` 注解 SQL** 为主 | 动态 SQL `<script>`、结果映射 |
| **MySQL 8 / 5.7** | 主机清单、任务、步骤日志、Agent 元数据 | `utf8mb4`、事务、索引、`init.sql` 迁移 |

**知识点：** 任务与 SSH 会话无状态；状态落库便于审计与 Web UI 分页查询。

### 3. SSH 远程执行

| 技术 | 用途 | 可延伸知识点 |
|------|------|----------------|
| **JSch（mwiede fork）** | `Session` + `ChannelExec` 在远端执行命令 | Known-hosts、密钥登录（可扩展）、超时与流读取模式 |

**知识点：** `exec` 通道需按 `available()`/`isClosed()` 轮询读取，避免 `BufferedReader.readLine()` 在无换行或未关流时永久阻塞。

### 4. 大模型与意图

| 技术 | 用途 | 可延伸知识点 |
|------|------|----------------|
| **OkHttp** | 调用 OpenAI **兼容格式** Chat API | 超时、Bearer Token、DashScope/Qwen 等 |
| **自然语言驱动** | 各 Agent 的 `system_prompt` 写在库表 `agent` 中（`linux_skills` 特例见下） | Prompt 约束为 JSON，便于后端解析 |

**知识点：** 异步任务可选用 `IntentRecognizer`，先关键词/rule，再兜底 LLM 选 Agent。

### 5. 异步与并发

| 技术 | 用途 | 可延伸知识点 |
|------|------|----------------|
| **Spring `@Async` + 线程池** | `AsyncTaskConfig`：`taskExecutor` 跑长任务 | 自调用不穿代理问题、线程池队列与拒绝策略 |
| **内存 `ConcurrentHashMap`** | `AsyncTaskService` 记录可取消上下文 | **取消**≠杀远端进程，仅中止本地编排循环为主 |

### 6. WebSocket（进度推送）

| 技术 | 用途 | 可延伸知识点 |
|------|------|----------------|
| **Spring WebSocket + STOMP + SockJS** | `WebSocketConfig`：`/ws`、主题 `/topic/task/{id}`、`/topic/tasks` | SockJS fallback、同源策略、大屏订阅 |

当前 **Web UI 静态页尚未订阅 STOMP**，但后端已预留广播 `TaskProgressBroadcaster`。

### 7. 安全与运维

| 要点 | 说明 |
|------|------|
| **CommandSecurityService** | `strict`/`permissive`/`disabled`、黑白名单、高危模式拦截 |
| **密码落库** | 示例为明文，生产应对 `server.password` 加密或改用 SSH 密钥 |
| **课设脚本 linux_skills** | Java 对用户输入做单引号转义后传给 `bash ... agent-router.sh '...'`，路由层不注入裸露 shell 元字符 |

### 8. 前端（内置控制台）

| 技术 | 用途 |
|------|------|
| **静态 HTML + CSS + Fetch** | `src/main/resources/static/`，无需 Node 打包即可随 jar 分发 |

│   │   ├── SshService + impl       # SSH 封装
│   │   ├── LlmService               # Chat 调用

│   │   ├── IntentRecognizer
│   │   ├── CommandSecurityService

## 三、各模块用途简述

| 模块 | 用途 |
|------|------|
| **ApiController / SystemController** | 对外 REST；任务同步/异步/工作流；主机 CRUD 与连通性测试；诊断；意图探测；skills 目录 JSON。 |
| **TaskOrchestrator** | 选择 `agent_type` → `plan()` → SSH → 写入 `task_log` → `nextStep()` 直到完毕或上限。 |
| **MultiAgentCoordinator** | 复杂指令由 LLM 产出「多步、多 Agent」计划，并按依赖调度；每步再走单 Agent 子循环。 |
| **LinuxSkillsAgent** | **不调用 LLM**；拼接 `bash '/path/in/linux/bin/agent-router.sh' '<用户句子>'`。 |
| **IntentRecognizer** | `auto` 模式：关键词 / 与 router 对齐的规则 → 失败则 LLM 选 `agent_type`。 |
| **AsyncTaskService** | 异步执行、取消标记、进度广播钩子。 |
| **SshServiceImpl** | 建连、执行、流读取与超时，供所有需 SSH 的路径复用。 |
| **static 控制台** | 浏览器内完成：加主机、SSH 测试、诊断、选 Agent 下发任务、看任务列表与详情。 |

## 四、使用方法（操作向）

### 1. 环境准备

1. **JDK 17**、**Maven**、**MySQL**（执行 `sql/init.sql`）。  
2. 修改 **`src/main/resources/application.yml`**：`spring.datasource.*`、`llm.*`。  
3. （可选）**linux_skills**：在 **Linux 目标机** 部署 `linux-agent-skills` 至 **`/opt/linux-agent-skills`**（或改 `linux-skills.remote-path` / `LINUX_SKILLS_REMOTE_PATH`）。

### 2. 构建与启动

```bat
mvn -DskipTests package
java -jar target\devops-agent-1.0.0.jar
```

或双击 **`start-devops-agent.bat`**（需已存在 `target\devops-agent-1.0.0.jar`）。

### 3. Web 控制台

浏览器打开：**`http://localhost:8080/`**  
在页面中：添加主机 → SSH 测试 → 选择 Agent 与说明文字 → 同步或异步执行 → 点击任务行查看日志。

### 4. API 前缀

所有业务接口在 **`http://localhost:8080/api/...`**，与 README 表格一致。

### 5. 典型选型

| 场景 | Agent / 路径 |
|------|----------------|
| 自然语言单次/多轮命令（需 LLM） | `command_executor` |
| 日志/输出分析结论 | `log_analyzer` |
| 容器/部署编排类描述 | `deploy_agent` |
| 课设 runbook：磁盘/networkd/systemd/日志等关键词 | **`linux_skills`**（远端需脚本） |
| 复杂多步 | **`POST /api/tasks/workflow`** 或控制台「多 Agent 工作流」 |

### 6. 部署课设脚本到 Linux（摘要）

```bash
sudo mkdir -p /opt/linux-agent-skills
sudo cp -r /path/from/repo/linux-agent-skills/* /opt/linux-agent-skills/
sudo chmod +x /opt/linux-agent-skills/bin/agent-router.sh /opt/linux-agent-skills/runbooks/*.sh
```

**无需**在 Linux 桌面上手动双击 `agent-router.sh`；Java 会通过 SSH **自动**执行上述 `bash`。
