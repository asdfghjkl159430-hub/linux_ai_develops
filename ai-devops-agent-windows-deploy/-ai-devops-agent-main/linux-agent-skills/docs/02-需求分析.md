# 需求分析

## 1. 背景

Linux 日常运维中大量问题可通过「固定命令组合 + 经验规则」定位：磁盘满、日志异常关键字、负载飙高等。课程推荐 **AI Agent Skills** 方向，本项目将「Agent」落为 **可解释的 Shell 编排器**：不训练大模型，而强调 **命令白名单、管道组合与可追溯日志**。

## 2. 用户角色

- **运维实验用户**：输入简短中文意图，获得终端诊断摘要。  
- **课程答辩教师**：关注 Linux 技术使用、自动化流程、测试与文档。

## 3. 功能需求

1. **FR-1 磁盘巡检**：展示文件系统类型、使用率、inode；对高使用率给出 WARN。  
2. **FR-2 日志审计**：对给定或默认日志路径做 ERROR/FAIL 等关键字扫描与计数。  
3. **FR-3 进程健康**：展示负载、Top 内存进程、可选 vmstat、systemd 失败单元。  
4. **FR-4 Agent 路由**：从自然语言映射到上述能力之一，并保存运行日志。  
5. **FR-5 安全执行**：Python 侧提供白名单执行样例，防止任意命令注入。  
6. **FR-6 Web 控制台**：浏览器输入意图，经 HTTP JSON 调用后端，由服务端 `subprocess` 触发与 CLI 相同的路由脚本并回显输出。

## 4. 非功能需求

- **NFR-1 可移植性**：优先 POSIX/Bash，命令缺失时降级。  
- **NFR-2 安全性**：默认不调用交互式 sudo；仅环境变量显式开启且 `sudo -n` 成功才提升读日志。  
- **NFR-3 可维护性**：`runbooks` 与 `bin` 分离，`lib/common.sh` 复用。  
- **NFR-4 可测试性**：提供 `tests/run_all_tests.sh` 与临时日志用例。

## 5. 约束

- 运行于 Linux；推荐 Ubuntu 系。  
- 不依赖 Docker/数据库（可选扩展）。
