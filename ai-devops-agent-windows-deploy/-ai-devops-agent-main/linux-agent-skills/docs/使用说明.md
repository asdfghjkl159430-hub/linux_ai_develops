# 软件使用说明

## 1. 环境要求

- Linux（推荐 Ubuntu 22.04+）  
- Bash 4+、`coreutils`、`grep`、`gawk` 或 `awk`、`procps`  
- 可选：`vmstat`（`sysstat`）、`findmnt`（`util-linux`）、`systemd`  

## 2. 获取代码

```bash
cd /opt/kcsj/linux-agent-skills
git status
```

## 3. 一键赋予执行权限

```bash
chmod +x bin/agent-router.sh runbooks/*.sh tests/run_all_tests.sh scripts/export_report.sh
chmod +x lib/safe_exec.py
```

## 4. 运行方式

### 4.1 Agent 路由（主入口）

```bash
# 磁盘
./bin/agent-router.sh "检查磁盘空间和 inode"

# 日志（默认尝试 /var/log 下常见文件；无权限则跳过或见下节）
./bin/agent-router.sh "分析系统日志里的 ERROR"

# 进程
./bin/agent-router.sh "看看进程负载和 systemd 失败服务"
```

### 4.2 仅运行某个 runbook

```bash
./runbooks/disk_check.sh
AGENT_USE_SUDO=0 ./runbooks/log_audit.sh /tmp/my-test.log
./runbooks/proc_health.sh
```

### 4.3 最高读权限模式（课设演示）

当需要读取当前用户不可见的系统日志，且实验机已为账号配置 **免密 sudo**：

```bash
export AGENT_USE_SUDO=1
./bin/agent-router.sh "审计 auth 日志错误"
```

> 警告：免密 sudo 具有安全风险，仅限教学实验环境；生产环境请使用专用只读账号或集中式日志。

### 4.4 Python 白名单执行

```bash
python3 lib/safe_exec.py list
python3 lib/safe_exec.py run -- df -h
```

## 5. 测试

```bash
./tests/run_all_tests.sh
```

## 6. 定时任务示例

见 `docs/crontab.example`，复制到 `crontab -e` 后修改路径。

## 7. Web 交互界面

```bash
cd /opt/kcsj/linux-agent-skills
chmod +x scripts/start_web.sh
./scripts/start_web.sh
```

浏览器打开 **`http://127.0.0.1:8765/`**（端口可用环境变量 `AGENT_WEB_PORT` 修改）。

- 在文本框输入自然语言，例如「检查磁盘空间」「分析系统日志错误」「看看进程负载」。  
- **仅干跑**：勾选后不真正执行 runbook，只打印将执行的脚本路径。  
- **提升读日志**：勾选后设置 `AGENT_USE_SUDO=1`（需本机免密 `sudo`）。  

后端通过 `POST /api/run` 将 `query` 传给 `bin/agent-router.sh`；**禁止** `;` `` ` `` `$` `|` 等 shell 元字符，防止注入。

## 8. 报告导出（可选）

若已安装 `pandoc`：

```bash
./scripts/export_report.sh
```

## 9. 常见问题

| 现象 | 处理 |
|------|------|
| `/var/log/auth.log` 无法读 | 使用测试日志路径；或 `AGENT_USE_SUDO=1` |
| `vmstat` 不存在 | 自动跳过，不影响其它输出 |
| 路由提示未识别意图 | 在语句中包含「磁盘」「日志」「进程」等关键词 |
