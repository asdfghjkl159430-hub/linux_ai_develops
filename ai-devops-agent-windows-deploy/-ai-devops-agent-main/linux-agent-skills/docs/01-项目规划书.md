# 项目规划书（简化版 · ≤3 页等效）

## 1. 项目名称

**Linux 运维 Agent Skills：巡检与诊断技能包**

## 2. 项目目标

在 Linux 环境下实现一套可运行的「Agent 执行逻辑」原型：将自然语言运维意图映射为受控 Shell 流水线（runbook），完成磁盘、日志、进程等常见巡检；满足课程对 **Shell、文本处理、管道、进程/日志、Git、测试与文档** 的综合要求，并体现 **AI Agent Skills** 方向中的「工具调用 + 自动化编排」。

## 3. 项目功能

| 模块 | 功能说明 |
|------|----------|
| 磁盘巡检 | `df`/`inode`/`findmnt` 与阈值提示 |
| 日志审计 | `grep`/`awk`/`tail` 关键字扫描与计数，可选 sudo 提升读权限 |
| 进程健康 | `uptime`/`ps`/`vmstat`/`systemctl --failed` |
| Agent 路由 | 关键词匹配 → 调用对应 runbook，结果 `tee` 落盘 |
| Python 白名单 | `safe_exec.py` 限制可执行命令集合 |
| Web 控制台 | Flask 页面输入意图 → 调用 `agent-router.sh` → 展示输出 |

## 4. 使用技术（对照课程「最低四项」）

- Bash Shell、管道与重定向  
- `grep` / `sed` / `awk`  
- 进程与日志分析、`/proc` 读取  
- **Git** 版本管理、**crontab** 示例（见 `docs/crontab.example`）  
- **Python 3** `subprocess` 白名单执行  
- **Agent Skills**：`skills/*/SKILL.md` 规格 + `bin/agent-router.sh` 编排

## 7. 可能风险与对策

| 风险 | 对策 |
|------|------|
| 无权限读 `/var/log/*` | 默认跳过；`AGENT_USE_SUDO=1` + 免密 sudo 用于演示；或指向可读测试日志 |
| 环境缺少 `vmstat`/`findmnt` | 脚本检测命令存在性并降级 |
| 误用 `sudo` 带来安全风险 | 仅对 `cat`/`tail` 读文件；文档强调演示机使用 |
